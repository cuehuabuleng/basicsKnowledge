var count let 的区别

1、使用var声明的变量都会挂载到window上面，而let和count不会
2、var声明变量存在变量提升，let和count不会
3、let和count声明形成块级作用域
4、同一作用域下面，let和count不可以声明相同名字的变量
5、count一旦声明就需要赋值，而且不可以使用null来占位，声明之后不能修改，但是如果是复合类型数据就可以改变他的属性。


为什么var可以声明重复的变量，是因为编译器会在作用域判断这是重复声明变量的时候，忽略var直接赋值。

说一下js代码运行是三者的作用：
js引擎：负责代码的编译和运行
编译器：负责词法分析、语法分析、代码生成等工作；
作用域负责维护所有的标识（变量）

```javascript
    var a = 2;
    var a = 3;
    a = 4;
    alert(a); // 4
```

重复声明时：首先，编译器对代码进行分析拆解，当遇见var a，则询问作用域是否已存在叫a的变量，若不存在，则要求作用域声明一个新的变量a，若已存在，则忽略var继续向下编译。
赋值时：引擎遇见a=2时同样会询问在当前的作用域下是否有变量a。若存在，则将a赋值为2；若不存在，则顺着作用域链向上查找，若最终找到了变量a则将其赋值2，若没有找到，则招呼作用域声明一个变量a并赋值为2（这就是为什么第二段代码可以正确执行且a变量为全局变量的原因，当然，在严格模式下JS会直接抛出异常：a is not defined）。
